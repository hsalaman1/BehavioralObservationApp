# Electron Desktop App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wrap the existing React+Vite app in Electron so it ships as a Windows `.exe` installer, while keeping all existing functionality and the web build intact.

**Architecture:** Electron's main process (`electron/main.js`) opens a `BrowserWindow`. In dev it loads the Vite dev server at `localhost:3000`; in production it loads the Vite-built `dist/index.html` via `loadFile`. A separate `vite.config.electron.js` builds the renderer with `base: './'` and no PWA plugin, so asset paths resolve from `file://`. electron-builder packages everything into a Windows NSIS installer.

**Tech Stack:** Electron (main process, ESM), electron-builder (NSIS packaging), concurrently + wait-on (dev orchestration), Vite (renderer build), React 18 (no changes)

## Task 4: Create `electron/main.js`

**Files:**
- Create: `electron/main.js`

- [ ] **Step 1: Create the main process file**

Create `electron/main.js`:

```js
import { app, BrowserWindow } from 'electron'
import { fileURLToPath } from 'url'
import path from 'path'

const __dirname = path.dirname(fileURLToPath(import.meta.url))

function createWindow() {
  const win = new BrowserWindow({
    width: 1280,
    height: 900,
    show: false,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      contextIsolation: true,
      nodeIntegration: false,
    },
  })

  if (app.isPackaged) {
    win.loadFile(path.join(__dirname, '../dist/index.html'))
  } else {
    win.loadURL('http://localhost:3000')
  }

  win.once('ready-to-show', () => win.show())
}

app.whenReady().then(createWindow)

app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') app.quit()
})

app.on('activate', () => {
  if (BrowserWindow.getAllWindows().length === 0) createWindow()
})
```

Key decisions:
- `app.isPackaged` is `true` only when running from an installed/built app — more reliable than `NODE_ENV`
- `show: false` + `ready-to-show` prevents a white flash on launch
- `contextIsolation: true` / `nodeIntegration: false` — Electron security best practice
- `__dirname` reconstructed via `fileURLToPath` because the package uses `"type": "module"` (ESM)

  plugins: [react()],

    emptyOutDir: true,
  },
  server: {
    port: 3000,
  },
})
```

> `vite.config.js` (the web build) is **not modified** — it keeps `base: '/'` and PWA intact.

  - electron/**

  target:
    - target: nsis
      arch:
        - x64

nsis:
  oneClick: false
  allowToChangeInstallationDirectory: true
  createDesktopShortcut: true
  createStartMenuShortcut: true
```

Notes:
- `files` includes the Vite-built renderer (`dist/**`), the main process (`electron/**`), and `package.json` (electron-builder requires it to find `"main"`)
- `node_modules` is NOT listed — the main process only imports from `electron` and Node builtins, both provided by the Electron runtime
- NSIS produces a standard Windows installer with optional install directory and shortcuts

## Notes for future builds

To rebuild the installer after code changes:

```powershell
npm run electron:build
```

To rebuild with updated Supabase credentials, update `.env` first (credentials are baked in at build time by Vite's `import.meta.env`).

To add a new Electron native feature (tray icon, native menus, IPC):
- Main process code goes in `electron/main.js`
- Expose APIs to the renderer via `electron/preload.js` using `contextBridge.exposeInMainWorld`
